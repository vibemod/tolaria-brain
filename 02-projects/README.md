# 02-projects/

Active efforts with a defined end state — something you're trying to finish.

Examples: ship the Q3 report, plan the kitchen renovation, hire a data engineer, prepare the conference talk.

One folder or file per project. When a project is done or abandoned, move it to `60-archive/` (prefixed `YYYY-MM-`).

Test: "done by when?" has an answer. If it never ends, it's an area — see `../03-areas/`.

Start every project note with this header so status is scannable at a glance:

```markdown
# Project name

- **Goal:** the finish line, in one sentence
- **Deadline:** 2026-09-30
- **Status:** active | waiting | stalled
- **Next action:** the single next concrete step
```
